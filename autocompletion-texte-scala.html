Prog Fonctionnel — Autocomplétion texte

Petit programme pédagogique en Scala 3 : à partir de textes littéraires français (Les Misérables, extraits Wikipedia), il construit un modèle de langue Markov à ordre fixe (n-grammes), puis stocke les probabilités du mot suivant dans un trie (structure imposée, clés = chaînes de contexte complètes). Deux usages en ligne de commande après sbt run : suggestions interactives (top 5) ou génération répétée du meilleur suivant (greedy). Pas de serveur HTTP ni interface web dans le dépôt.

Programmation fonctionnelle • Scala 3.3.7 • sbt
Dépôt GitLab

Enchaînement automatique

  • fusion des fichiers .txt sous src/main/resources/ ;
  • tokenisation (minuscules, lettres françaises, tirets coupés) ;
  • apprendre(texte, n) — fenêtre glissante, comptes puis normalisation (MLE) ;
  • modeleVersTrie — construction du trie de Distribution ;
  • mode 1 : boucle terminale « phrase + suggestion » ; mode 2 : génération jusqu’à N mots ou arrêt si contexte inconnu.

Les sorties peuvent paraître plausibles au niveau des enchaînements lexicaux, sans cohérence sémantique — comme prévu dans l’extension « aller plus loin » du sujet.

Comment lancer

# À la racine du projet sbt « Prog Fonc - Projet Autocompletion Texte »
sbt run

L’outil demande la taille de contexte n (entrée vide = 3 par défaut), puis le mode (autocomplétion ou génération). Entrées et sorties passent par le terminal (StdIn / impressions). Après clonage depuis le dépôt GitLab, exécute ces commandes à la racine du projet.

Limitations (transparence)

  • Pas de lissage ni de backoff — contextes rares ⇒ pas de prédictions.
  • Génération greedy, déterministe, pas de tirage probabiliste ni de « petit LM » type réseaux de neurones.
  • Bruit mineur possible (en-têtes Gutenberg en anglais encore tokenisées).
